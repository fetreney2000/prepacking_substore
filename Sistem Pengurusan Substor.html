Bantuan

Pengenalan

Sistem Inventori Prabungkus adalah aplikasi pengurusan inventori untuk Substor Hospital Keningau. Sistem ini membantu pengurus farmasi menjejaki stok ubat prabungkus, mengira tahap inventori optimum, membuat pesanan pembelian, dan menjana laporan.

Papan Pemuka (Dashboard)

Paparan utama menunjukkan ringkasan inventori:

  • Jumlah SKU Aktif — Bilangan item aktif dalam sistem
  • Kumpulan — Bilangan kumpulan item
  • Jumlah Stok Semasa — Jumlah keseluruhan stok semua item
  • Stok Rendah/Kehabisan — Bilangan item yang perlu dipesan
  • Jadual Stok Rendah — Senarai item yang perlu dipesan

Pengurusan SKU

Halaman ini membolehkan anda mengurus item inventori:

  • Tambah SKU — Klik butang "Tambah SKU" untuk menambah item baharu. Isi kod, nama, saiz pek, kumpulan, dan data penggunaan.
  • Sunting — Klik ikon pensel untuk mengubah suai maklumat item.
  • Padam — Klik ikon tong sampah untuk memadam item (tidak boleh jika masih ada dalam pesanan).
  • Cari — Gunakan medan carian untuk mencari mengikut kod atau nama.
  • Tapis Kumpulan — Pilih kumpulan untuk menapis senarai.
  • Tukar Stok — Klik pada nilai stok semasa untuk mengedit secara langsung.
  • Pilihan Kolom — Gunakan suis togol untuk menunjuk/menyembunyikan lajur.

Medan Penting:

  • Kod — Kod unik item (contoh: C08CA01000T1002XX)
  • Saiz Pek — Bilangan unit dalam satu pek
  • Penggunaan Bulan 1/2/3 — Jumlah penggunaan untuk 3 bulan terakhir
  • Sentiasa Stok Penuh — Aktifkan untuk item yang sentiasa ditopup ke stok maksimum
  • Guna Tahap Manual — Aktifkan untuk menetapkan tahap Min/Penimbal/Maks secara manual
  • Aktif — Nyahaktifkan item yang tidak lagi digunakan

Kumpulan

Kumpulan mengorganisasikan item mengikut kategori. Contoh: APPL (Tablet & Kapsul), CFLNS (Sirap), Bukan Ubat. Setiap SKU perlu dikaitkan dengan satu kumpulan.

Cipta Pesanan

Untuk membuat pesanan baharu:

  • Tarikh — Tarikh pesanan (auto-diisi dengan tarikh hari ini)
  • Nama Pembuat — Nama orang yang membuat pesanan
  • Tempoh (Minggu) — Tempoh bekalan dalam minggu. Ubah nilai ini untuk melihat kuantiti pesanan dikira semula secara automatik.
  • Nota — Nota tambahan untuk pesanan

Jadual item akan menunjukkan semua item aktif dengan kuantiti pesanan yang dikira secara automatik berdasarkan AWU × Tempoh − Stok Semasa. Anda boleh mengubah kuantiti secara manual. Tetapkan kuantiti kepada 0 untuk item yang tidak perlu dipesan.

Edit Pesanan

Pilih pesanan dari senarai untuk mengubah suai tarikh, nota, atau item. Anda boleh menambah item baharu, mengubah kuantiti, atau memadam item. Kuantiti item baharu dikira automatik berdasarkan tempoh pesanan.

Laporan Pesanan

Menjana laporan pesanan berdasarkan penapis:

  • Tarikh Dari/Hingga — Tapis mengikut julat tarikh
  • Nama Pembuat — Tapis mengikut nama pembuat
  • Item — Tapis mengikut item (SKU) tertentu
  • Jana Laporan — Klik untuk menjana laporan
  • Pilihan Kolom — Togol lajur untuk ditunjuk/disyembunyikan

Cetak: Klik butang "Cetak" pada senarai pesanan untuk membuka halaman cetak dalam tetingkap baharu dengan format "Borang Permohonan Stok".

Laporan SKU

Menjana laporan status stok mengikut kumpulan:

  • Status — Tapis mengikut status: Kritikal, Amaran, Baik, atau Kehabisan
  • Jana Laporan — Klik untuk menjana laporan
  • Cetak — Cetak laporan dalam tetingkap baharu
  • Pilihan Kolom — Togol lajur (Min, Penimbal, Maks, Kumpulan tersembunyi secara default)

Pengiraan Tahap Inventori

  • AWU (Penggunaan Mingguan Purata) = (Bulan 1 + Bulan 2 + Bulan 3) ÷ 12
  • Min = AWU × Minggu Minimum (dibulatkan ke atas mengikut saiz pek)
  • Penimbal = AWU × Minggu Penimbal (dibulatkan ke atas mengikut saiz pek)
  • Maks = AWU × Minggu Maksimum (dibulatkan ke atas mengikut saiz pek)

Nilai Minggu Minimum, Penimbal, dan Maksimum boleh ditetapkan di halaman Tetapan.

Status Stok

  • OK Stok melebihi tahap penimbal — stok mencukupi
  • Rendah Stok antara min dan penimbal — perlu dipesan
  • Kritikal Stok di bawah min — perlu dipesan segera
  • Kehabisan Stok sifar

Tetapan

Konfigurasikan parameter pengiraan: tajuk aplikasi, minggu minimum, penimbal, dan maksimum. Mod paparan boleh ditukar antara Jadual dan Kad.

Penyelarasan (Sync)

Pengurusan data:

  • Simpan — Muat turun fail JSON mengandungi semua data (tetapan, kumpulan, SKU, pesanan, item pesanan)
  • Import — Muat naik fail JSON untuk menggantikan semua data sedia ada (berhati-hati: ini akan memadam data sedia ada)
  • Import Excel — Muat naik fail Excel (.xlsx) untuk mengemas kini stok semasa mengikut kod item

Pilihan Kolom Jadual

Semua jadual mempunyai suis togol di atasnya. Gunakan suis ini untuk menunjuk atau menyembunyikan lajur mengikut keperluan. Keadaan lajur ditetapkan semula setiap kali halaman dimuat semula.
